大二学生用 Agent + Obsidian 搭建 OPC 一人公司，半年变现 6 万+

大二学生用 Agent + Obsidian 搭建 OPC 一人公司，半年变现 6 万+

灿烂的番茄2026-07-212026-07-23

前言

这两年 AI 技术发展太快了，各种新词层出不穷：Agent、知识库、工作台、Skills、一人公司……听起来都很厉害，但普通人到底怎么用它们赚钱？

最近读到一位大二学生的案例，他叫安十一，从去年 10 月开始接触 AI，借助 Agent + Obsidian 搭建了一套 OPC（One Person Company，一人公司）赚钱系统，大半年累计变现 6 万+。他不是计算机专业，也不是程序员，纯靠自学 AI 提示词、AI 设计、AI 编程、课程制作，以及 Agent + 知识库相关的能力，硬生生从零跑通了一条路。

这个收入对于工作多年的职场人来说不算大钱，但对于一个 20 岁的在校生来说，已经秒杀绝大多数同龄人。更值得关注的是他的系统化思路——他不是在”做项目”，而是在”建系统”。这篇文章把他的整套方法论拆解出来。

OPC 不是让你一个人干十个人的活

很多人听到”一人公司”，第一反应是：一个人注册一家公司，一个人做老板，一个人把所有工作全干完。

但安十一的理解不一样。他认为 OPC 的核心不是”让自己活成十个人”，而是一套围绕一个人运转的小型商业系统。这套系统只有三个环节，但缺一不可：

前端获客：让对的人先看到你

前端获客分两条线同时走。

第一条线是内容建立认知。通过公众号和知乎持续输出文章，让潜在用户慢慢了解：你是谁，你长期在研究什么，你能解决什么问题。这些文章不一定发出去就有人买单，但它们在慢慢积累信任资产。

第二条线是产品作为入口。在小红书、闲鱼等平台上架虚拟产品，比如 AI 学习手册、教程、数字产品。用户可能根本不知道你是谁，只是先看到你的产品，觉得能解决他的问题，于是购买、进入你的私域，再慢慢了解你。

两条线分工不同：公众号和知乎偏向建立认知和信任，小红书和闲鱼更像具体的产品入口。

中端转化：不是推销，是持续展示

前端把人带过来之后，中间还需要建立信任。这一步最容易被误解。

安十一的做法是：用户添加微信之后，通过朋友圈、社群分享、课程案例、学员反馈，持续展示自己每天在做什么。用户慢慢看到——这个人不是只会讲概念，他确实长期在做 AI，也确实做过不少真实项目。等用户真正有需求时，自然会购买课程、预约咨询或邀请培训。

中端转化的核心不是加上微信就硬推销，而是通过持续输出，让用户慢慢知道你是谁、你能干什么、为什么值得信任。

后端交付：两条变现线互相喂养

后端就是用户付钱之后，你交付什么。安十一的后端有两条明确的变现线：

第一条：做课卖课。 把已经跑通的方法论做成课程，规模化复制。课程是标准化产品，一次制作可反复出售，边际成本极低。他的逻辑很务实：先做市场调研，判断方向有没有真实需求、自己能不能快速掌握，确认之后再进入课程制作阶段。大纲由他和 Agent 协同完成，他负责确定目标用户和核心知识，Agent 辅助整理章节结构，最终交付为飞书、网页或 PDF 形式的课程手册。

第二条：一对一咨询和企业培训。 这条线单价更高，主要面向中小企业老板和管理层。不是拿一套固定课程反复讲，而是先了解客户的业务现状、卡点和 AI 基础，再定制学习路径。比如做美容店的客户，重点教 AI 提示词和设计；管十几家门店的客户，偏重内容制作和知识管理。

关键洞察是：这两条线在互相喂养。课程解决规模化，把跑通的方法复制给更多人；咨询解决深度，逼着你不断遇到真实问题、持续迭代。课程学员遇到个性化问题会转化为咨询客户，咨询中反复出现的场景又可以整理成新课程。一个负责复制，一个负责生长。

Agent + Obsidian：你的 AI 第二分身

OPC 的结构清楚了，但一个人怎么同时搞定获客、转化和交付？这里就需要 Agent 和 Obsidian 配合出场。

Agent 能干什么，不能干什么

Agent 不只是聊天机器人，而是真正能帮你干活的 AI。比如 Claude Code、Codex、WorkBuddy 这些工具，可以读文件、整理资料、改内容、生成文档，甚至搭建工作流。

但 Agent 有个大问题：它不了解你。每次开新任务都要重新告诉它——我是谁？我的风格是什么？哪些能改哪些不能改？说一次还好，次次说就变成了”重复培训 AI”。

Obsidian 解决 Agent 的失忆症

Obsidian 本质上是一个本地知识库，但安十一不是把它当普通笔记用，而是把它设计成一套能被 AI 理解、调用、执行并持续更新的个人工作系统。他的知识库分为五个相互配合的模块：

第一部分：Agent 入口层。 专门写给 AI 看的说明书和导航地图。告诉 AI 知识库的结构、每个模块负责什么、遇到不同任务时去哪找资料。这样 AI 不需要每次读完整个知识库，而是先了解结构再按需读取。

第二部分：个人 IP 中控台。 沉淀”我是谁”。记录个人定位、成长经历、写作风格、产品体系、真实案例和判断标准。AI 读取这些内容后才知道按谁的方式工作，写出来的文章更像你，做出来的课程也能结合你的真实经历。

第三部分：知识库生长系统。 规定新资料放哪、哪些内容需要审核、新旧信息冲突怎么处理、每次更新如何留记录。知识库不是搭完就不变了，它要随着业务一起成长。

第四部分：工作流与 Skills 层。 每个 Skill 是一份给 AI 的固定工作说明书，比如公众号写作、课程大纲制作、学习手册制作、图片规划、朋友圈创作等。每个 Skill 只负责一类任务，写清楚要读什么资料、按什么步骤执行、输出什么结果。

第五部分：工作流指挥系统。 真实工作通常需要多个 Skills 配合。比如做一门课程，要依次调用资料分析、大纲生成、写作风格、学习手册、配图规划等模块。指挥系统负责判断任务类型、选择 Skills、安排执行顺序，把多个模块串成完整流程。

当他把一个任务交给 Agent 后，Agent 会先读入口说明，再调用个人 IP 资料，根据任务选择对应 Skills，由指挥系统统一安排执行。任务完成后，新内容还会按生长规则沉淀回知识库。

具体能做什么

这套系统跑起来之后，已经在帮他做三件具体的事：

写文章。 他先用语音输入把脑子里的想法说出来，得到一版比较乱的草稿（有重复、有口语、逻辑没整理清楚）。然后交给 Agent，Agent 读取他的写作风格规范、个人经历和内容创作流程，整理成一篇完整的文章。关键不是”让 AI 写文章”，而是让 AI 按照他的风格、他的经历去整理。

做朋友圈内容。 他把自己的定位、产品、课程、案例和近期工作持续记录到知识库。需要写朋友圈时，Agent 直接读取这些信息，知道他最近在做什么、有什么产品和案例，生成不同角度的内容，不再是每次从零编。

制作课程。 这是他使用最多的一套工作流。输入课程资料和案例后，Agent 按顺序处理：课程大纲 → 课程讲稿 → 学习手册 → 配图方案 → PPT 内容 → 交付资料。以前每做一门课都要重新想一次，现在是先搭好系统，再让 AI 按系统执行——看起来只是状态不同，实际效率完全不一样。

从追项目到建系统

安十一的经历还有一个值得讲的侧面：他是怎么从”追风口”转变到”建系统”的。

刚进大学时，他和大多数做互联网的人一样，最关心的问题就是”最近什么项目赚钱”。做过闲鱼虚拟、做过小红书虚拟，有些赚到了，有些交了学费。但慢慢发现短期项目有一个致命问题：它不稳定。平台规则一变，原来的玩法就失效了；某个产品的热度一过，销量马上掉下来。甚至有一次学习手册做到 85%，平台突然调整规则，整个路径就走不通了。

经历了这些之后他才真正想明白：做项目可以让你赚到一次钱，但只有建立自己的系统才能持续赚钱。他不再问”下一个赚钱项目是什么”，而是开始问自己——客户为什么认识我？流量最后沉淀在哪里？我有什么可以长期卖的产品？就算某一个平台发生变化，我还能不能继续运转？

这个心态转变，才是他真正开始搭建 OPC 的起点。

现状与展望

安十一坦诚地承认，这套系统还远谈不上成熟。公域流量还不够稳定，很多服务仍然依赖他本人，咨询和培训的经验也没有全部完成产品化。但他已经不再焦虑，因为他知道自己接下来应该做什么：

  1. 继续把已跑通的课程和交付流程标准化
  2. 进入更多真实的业务场景，积累真正能产生结果的案例
  3. 把咨询中反复出现的问题做成课程、模板、工作流

回头看，他以前一直在找能赚钱的项目——今天做闲鱼，明天做小红书，后天追新机会。这些经历没有白费，让他理解了流量、产品、私域和交付。但也让他意识到：只依赖某一个项目，你永远不知道它什么时候会停。他现在真正想拥有的不是下一个短期风口，而是一套真正属于自己的系统——里面有内容、私域、产品、客户、知识库和工作流程。

总结：OPC 的核心是人决定方向，AI 负责执行

这套系统最关键的一个洞察是：Agent + Obsidian 本身不会直接带来客户，真正产生收入的仍然是内容、产品、咨询、案例和用户信任。AI 的作用是让这些事情运转得更稳定。

角色 负责
人 方向、规则、判断、决策、最终审核
AI 整理、执行、复用、提效、放大交付能力

OPC 的核心不是你什么都会做，而是你能不能把自己的经验、规则和业务逐渐交给一套系统去协同执行。

普通人怎么开始？

不需要一上来就搭复杂的 Agent，也不需要建几十个知识库文件夹。先理清三件事：

  1. 你的客户从哪里来？（前端获客渠道）
  2. 你准备向他卖什么？（产品/服务形态）
  3. 你的工作中，哪件事最重复、最适合先交给 AI？（第一个自动化点）

先让一个真实客户愿意为你提供的一个真实结果付钱，然后把这件事一点点做成流程、做成系统、做成产品。这才是普通人开始搭建 OPC 最现实的一条路。
