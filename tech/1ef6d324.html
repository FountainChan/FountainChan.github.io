🔧 第一步：准备你的创作环境

在开始构建博客之前，你需要先在本地电脑上准备好必要的工具。

  1. 安装 Node.js 和 npm
    Hexo 基于 Node.js，所以首先需要安装它。请访问 Node.js 官网，下载并安装 LTS（长期支持版） 。安装程序会同时包含 npm（Node.js 的包管理器）。

  2. 安装 Git
    Git 是版本管理工具，也是与 GitHub 通信的桥梁。请访问 Git 官网下载并安装。安装后，打开命令行（终端或 Git Bash），设置你的全局用户名和邮箱，这在后续的 Git 提交中会用到：

  3. （可选但推荐）配置 SSH 密钥
    为了避免每次与 GitHub 通信都输入密码，可以配置 SSH 密钥进行安全连接。

    • 生成密钥对：在命令行中运行 ssh-keygen -t rsa -C "你的邮箱"，连续按回车使用默认设置即可。
    • 添加公钥到 GitHub：登录 GitHub，进入 Settings > SSH and GPG keys，点击 “New SSH key”，将生成的 .pub 公钥文件内容粘贴进去。

📦 第三步：在 GitHub 上安放你的博客

你的博客内容将存放在两个 GitHub 仓库里：一个存放 Hexo 的源码，另一个由 GitHub Actions 自动生成，用于存放编译好的静态网页。

  1. 创建源码仓库（Source Repository）
    这个仓库用于保存你所有的博客文章、主题和配置文件。登录 GitHub，点击右上角 “+” 号，选择 “New repository”。

    • 仓库名：可以任意取名，例如 my-hexo-blog-source。
    • 权限：建议设置为 Private（私有） ，这样你的文章草稿和配置就不会公开。
    • 其他保持默认，点击创建。

  2. 创建页面仓库（Pages Repository）
    这个仓库将用来托管你的公开博客网站。同样创建一个新仓库。

    • 仓库名：必须严格遵守格式：<你的GitHub用户名>.github.io。例如，如果你的用户名是 johnsmith，仓库名就是 johnsmith.github.io。
    • 权限：必须设置为 Public（公开） ，否则 GitHub Pages 无法免费托管。
    • 点击创建。

🤖 第四步：配置自动化引擎 GitHub Actions

这是最关键的一步，我们将设置一个自动化流程：每当你把新的文章或修改推送到源码仓库时，GitHub Actions 会自动帮你构建网站，并部署到页面仓库。

  1. 创建 Workflow 文件
    在你的本地博客根目录（myblog 文件夹）下，手动创建一个新文件，路径为：.github/workflows/pages.yml。这个文件将定义自动化流程。

  3. 配置Github PAT
    使用 peaceiris/actions-gh-pages Action 进行跨仓库部署的完整配置。

    1. 创建 Personal Access Token (PAT)

      这是实现跨仓库推送的关键。

      • 访问 GitHub Token 设置页面。
      • 点击 Generate new token -> Generate new token (classic) 。
      • 填写 Note（如 Hexo Blog Deploy）。
      • 过期时间：建议设置为自定义，并选择一个较长的期限（例如1年）。
      • 权限范围：务必勾选 repo（包含所有仓库的完全控制权限）。
      • 点击生成，并立即安全地复制生成的 token 字符串。

    2. 在源码仓库配置 Secret

      • 进入你的私有源码仓库 SourceHexoBlog。
      • 点击 Settings -> Secrets and variables -> Actions。
      • 点击 New repository secret。
      • Name 输入：PERSONAL_ACCESS_TOKEN（必须与此名称完全一致）。
      • Value 粘贴你刚才复制的 token。
      • 点击 Add secret。

🌍 （可选）绑定自定义域名

如果你已经购买了域名，想要用像 blog.yourdomain.com 这样的地址访问博客，可以额外进行以下操作：

  1. 在博客源码的 source 目录下，创建一个名为 CNAME 的文件（没有后缀名），文件内容就是你的一级或二级域名，例如 blog.yourdomain.com。
  2. 在你的域名注册商（如阿里云、腾讯云）的管理后台，为这个域名添加一条 CNAME 记录，将其指向 你的用户名.github.io。
  3. 将 CNAME 文件随代码一起推送到源码仓库。之后，在页面仓库的 Settings > Pages 中，你还可以填写你的自定义域名。

💡 温馨提示

  • 核心配置：确保 pages.yml 工作流文件中的 node-version 与你本地环境的主要版本一致，这可以避免因环境差异导致的构建失败。
  • 主题注意事项：如果你使用的主题是以 Git 子模块（Submodule）方式安装的，Workflow 配置中已包含 submodules: recursive 选项来正确处理它。
  • 耐心等待：第一次部署和 DNS 解析可能需要一些时间，如果立即访问看不到变化，请耐心等待几分钟再刷新。
